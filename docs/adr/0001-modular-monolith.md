# ADR-0001. Модульный монолит

- **Статус:** принято (Фаза 0/1, 2026-07-06; источник — мастер-промпт v2, «Архитектурные решения» №1)
- **Контекст.** Шаблон обслуживает десятки небольших клиентских проектов рынка УЗ/СНГ силами соло-разработчика/микро-команды. Микросервисы дают операционную сложность (оркестрация, наблюдаемость, распределённые транзакции), которую этот масштаб не окупает.
- **Решение.** Один деплоймент — модульный монолит: ядро (core/), бизнес-модули (modules/), общий слой (shared/). Границы модулей — публичные сервисные интерфейсы и события шины; чужие таблицы не читаются никогда.
- **Последствия.** Любой модуль можно выпилить или вынести в отдельный сервис позже — границы уже проведены (интерфейсы + события, собственные таблицы, собственные миграции). Цена — дисциплина границ внутри одного процесса; она автоматизирована (import-linter, тест честности манифестов в Фазе 6).
