# ADR-0004. Трёхуровневый конструктор: ядро → модули → фичи

- **Статус:** принято (Фаза 0/1, 2026-07-06; источник — мастер-промпт v2 №4)
- **Контекст.** Из шаблона собираются десятки разных клиентских проектов; единицей переиспользования должен быть не «весь бэкенд», а отчуждаемый вертикальный срез.
- **Решение.** Три уровня: **ядро** (auth, tenants, billing, notifications, audit, admin — нужно каждому проекту), **модули** (предметные области: commerce, позже crm/saas; включаются `ENABLED_MODULES`), **фичи** (самодостаточные папки внутри модуля с манифестом `feature.toml`, переносятся копированием). Манифест — механизм, а не документация: автодискавери и проверка requires на старте, перенос цепочкой через tools/add-feature, тест честности манифестов в CI (Фаза 6).
- **Последствия.** Автодискавери (роутеры, admin-экраны, миграции собираются сканированием папок) — обязательное свойство всех фаз: клиентская фича = новая папка, merge шаблона её не трогает. Выключенный модуль физически не импортируется: его роутов, таблиц и тестов в проекте нет.
