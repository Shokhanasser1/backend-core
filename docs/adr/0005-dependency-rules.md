# ADR-0005. Правила связей: только «вниз», горизонталь — интерфейсы и события

- **Статус:** принято (Фаза 0/1, 2026-07-06; источник — мастер-промпт v2 №5)
- **Контекст.** Конструктор жив, пока фичи отчуждаемы; каждая скрытая связь (импорт чужих внутренностей, JOIN к чужой таблице) делает перенос фичи ложью.
- **Решение.** Зависимости направлены только вниз: `app → core → shared` (фичи → ядро → shared). Горизонтальные связи — ровно два способа: вызов публичного сервисного интерфейса (реэкспорт в `__init__.py` модуля) или подписка на события шины. **Чтение чужих таблиц запрещено всегда, без исключений.** Каждая горизонтальная связь фичи декларируется в feature.toml и проверяется автоматически.
- **Enforcement.** import-linter в CI (Фаза 1): контракт слоёв `app → shared` (расширяется на core в Фазе 2) + запрет `shared.system_repository` вне core. Тест честности манифестов — Фаза 6.
- **Последствия.** Публичные интерфейсы ядра — контракт; ломать их можно только MAJOR-релизом шаблона (ADR-0009).
